how this page works

Static HTML on GitHub Pages — no server rendering this page. Your message goes through a small MessageAdapter interface with two interchangeable backends:

firestore → a direct, rules-secured write to a Firestore collection (create-only; nothing on this page can read messages back).

aws → a POST to an API Gateway + Lambda endpoint that validates, rate-limits, and stores/forwards.

One config flag switches between them. No third-party form services, no trackers, no analytics scripts.
